# План реализации: «Офис в кармане» — платформа оркестрации ИИ-команд

**Краткое описание MVP:** Пользователь регистрируется, проходит онбординг, создаёт задачу текстом, ИИ-менеджер анализирует → задаёт вопросы (до 3 циклов) → генерирует план → пользователь утверждает план на канбан-доске из 6 колонок.
**Стек:** Next.js 15 (App Router) + TypeScript + Tailwind CSS + Prisma + SQLite (локально) / PostgreSQL Supabase (прод) + NextAuth.js + DeepSeek / OpenRouter (LLM, через Provider Adapter)
**Деплой:** Vercel (Hobby) + Supabase (бесплатный тир) → после MVP — индустриализация на российское облако
**Бюджет токенов LLM:** DeepSeek + бесплатные модели OpenRouter; на разработку ~$20-50/мес

## Этап I: Индустриализация — от Vercel-стартапа к промышленному масштабу

После полной реализации всех спринтов (MVP → Enterprise) продукт находится на Vercel + Supabase. Для промышленного масштаба (10K+ пользователей, 1000 задач/сек, 99.9% SLA) необходимо перейти на enterprise-архитектуру.

- [ ] **I.1** Миграция на российское облако / Kubernetes:
  - Переход с Vercel на VK Cloud / Yandex Cloud / собственный K8s-кластер.
  - Docker-контейнеризация всех сервисов (Next.js, воркеры, фоновые задачи).
  - Настройка Ingress, балансировщиков нагрузки.
  *Критерий:* `docker compose up` поднимает полное окружение (приложение + БД + Redis + воркеры). Продакшен работает в K8s.

- [ ] **I.2** Горизонтальное масштабирование:
  - Stateless Next.js (несколько реплик).
  - Воркеры для LLM-вызовов (KEDA автомасштабирование по длине очереди).
  - Шардирование БД по tenant_id (модель «база на тенант» для Enterprise).
  - Redis для кэширования эмбеддингов, сессий, очередей.
  *Критерий:* спайк-тест 10x RPS без падения. Автомасштабирование срабатывает <60 сек.

- [ ] **I.3** Мониторинг и наблюдаемость:
  - Prometheus + Grafana: дашборды RPS, latency (p50/p95/p99), ошибки, очереди.
  - OpenTelemetry / Jaeger: распределённый трейсинг каждого вызова LLM.
  - ELK / Loki: централизованные логи с индексацией по tenant_id, trace_id.
  - Алертинг: аномалии, ошибки 5xx, долгие ответы, переполнение очередей.
  *Критерий:* тестовая ошибка вызывает алерт в Telegram/email <2 мин. Дашборд Grafana отображает метрики в реальном времени.

- [ ] **I.4** Надёжность и отказоустойчивость:
  - Резервное копирование БД: инкремент каждые 15 мин, полное — 24ч.
  - RTO: 4ч (Business+), 24ч (Free/Pro). RPO: 15 мин (Business+), 1ч (Free/Pro).
  - Graceful degradation: при недоступности LLM задачи ставятся в очередь с уведомлением пользователя.
  - Multi-AZ развёртывание.
  *Критерий:* ручное отключение одного AZ — сервис продолжает работать. Восстановление из бэкапа <RTO.

- [ ] **I.5** CI/CD для промышленного окружения:
